Alright, let’s break this down cleanly and then give you the exact query.
🧠 What we need to compute

From your schema :

    A customer → linked to person

    A customer_order → linked to customer

    An order_details → contains:

        price_with_discount

        product_amount

👉 So order total =
price_with_discount * product_amount
⚙️ Steps logically

    Compute total per order

    Aggregate per customer:

        AVG(order_total) → avg_purchase

        SUM(order_total) → sum_purchase

    Filter:

        avg_purchase > 70

    Sort:

        sum_purchase ASC

        surname ASC

✅ Final SQL Query

WITH order_totals AS (
    SELECT 
        co.id AS order_id,
        co.customer_id,
        SUM(od.price_with_discount * od.product_amount) AS order_total
    FROM customer_order co
    JOIN order_details od ON od.customer_order_id = co.id
    WHERE co.customer_id IS NOT NULL
    GROUP BY co.id, co.customer_id
)

SELECT 
    p.name,
    p.surname,
    ROUND(AVG(ot.order_total), 2) AS avg_purchase,
    ROUND(SUM(ot.order_total), 2) AS sum_purchase
FROM order_totals ot
JOIN customer c ON c.person_id = ot.customer_id
JOIN person p ON p.id = c.person_id
GROUP BY p.name, p.surname
HAVING AVG(ot.order_total) > 70
ORDER BY sum_purchase ASC, p.surname ASC;

        You’d average individual items, not full orders ❌

    Then aggregate again per customer ✔️

                    )
            END
        , 2) AS difference

    FROM product p
    LEFT JOIN order_details od ON od.product_id = p.id
    WHERE p.id IS NOT NULL   -- dummy WHERE

    GROUP BY p.id, p.comment
) t
ORDER BY t.id ASC;
